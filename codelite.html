Codelite

L'interface propose plusieurs fonctionnalités assez simple a prendre en main, on trouve celle deja presente dans tout les ide: création de nouveau fichier et dossier, enregistrement, impression, undo et redo,rechercher, remplacer , etc.... avec la possibilité de les utiliser dans un Output Wiew avec d'autre fonctionnalités. Cette éditeur dispose de la coloration synthaxique, de l'auto-complétion, de plusieurs themes différents pour personnalisé l'apparence comme Monokai_2, roboticket, classic eclipse, etc... . Cet ide est completement configurable, l'ajout de différents plugin permet de configuré et de facilité sont utilisation, un live preview est aussi disponible ; il est possible de le configurer pour le debug du code en PHP. Les espaces de travail sont créer sous forme de dossiers avec une perspective(layout) qui permet de modifié son apparence une barre coloré est affiché sur le coté de l'éditeur lors de la frappe et permet de voir les lignes qui ont été modifié avant l'enregistrement cela ajoute un visuel des modifications. une fois enregistrer la barre lateral est de couleur uniforme (vert). un lien avec git permet aussi de push les codes vers un repository sans ouvrir de client comme smartgit ou meme directement sur Github(attention tout de même a cette fonctionnalité) pour finir, cette ide est assez simple a utilisé après l'avoir pris en main, les liens vers d'autres logiciels est un plus.

Les plugins

Git:

  • Git clone – clone un repositoire existant
  • Apply patch -applique un correctif rapide
  • Add file – Ajoute un fichier non pisté au répositoire local, ou « stage » un fichier mofifié
  • Reset file – Annule tout les changements effectués sur un fichier en local.
  • Refresh view – Equivalent de la ligne de commande 'git status -s'
  • Reset repository – Equivalent de la ligne de commande 'git reset --hard'
  • Pull remote changes - pull les modifications sur le repositoire distant et effectue (facultativement) une rebase
  • Commit local changes – commit les changements sur le repositoire git local
  • Push local changes - push les modifications locales commitées et les envoie sur le repositoire distant
  • Créée une branche locale, changé de branche local
  • Show current diff – Affiche une liste de fichiers modifiés dans son repositoire et visualise les modifications dans la diff view
  • Browse commit history – Ouvre une fenêtre avec une liste comprenant l’ensemble des commits effectués . En addition on peut faire un clique droit pour visualiser les dits-commits avant de potentiellement les récupérer comme base de travail.

SnipWiz 

Permet d’insérer rapidement des snippets de codes communs dans son travail. Plusieurs sont fournis par défaut, mais on peut lesmodifier, en ajouter ou en retirer à sa guise. Il est possible d’insérer des snippets en faisant un clic droit et en sélectionnant SnipWizz dans le menu contextuel.

Il y a 2 symboles « Occupants » disponibles :

  • $ sera remplacé par la sélection si elle existe.
  • @ positionne le pointeur une fois le snippet placé.
